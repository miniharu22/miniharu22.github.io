Very Deep Convolutional Networks for large-scale image recognition

[참고 논문 링크](https://arxiv.org/abs/1409.1556)

## 0. Architecture  

&nbsp;

  <p><em></em></p>
</div>

&nbsp;

- **Convolution Layer**   
  - 학습 과정에서는 224 x 224 size의 이미지가 입력으로 주어짐   
    - 전치리 방법은 각 이미지의 pixel에서 평균 RGB 값을 빼주어 정규화를 시켜준 것이 유일  
  - 전처리를 거친 이미지는 여러 개의 Convolution Layer를 거치게 되며 이때 각 Layer에서는 다음 2가지 종류의 kernel들만 사용   
    - **3 x 3 kernel** : 상하좌우의 인접한 pixel들을 인식하기 위한 최소 크기의 kernel  
    - **1 x 1 kernel** : 입력 채널에 대한 선형 변환으로 해석됨 (stride : 1)
  - 각 Convolution Layer에서 Padding은 이전의 입력으로 들어온 이미지의 size를 유지할 수 있을만큼 설정    
  - Max Pooling은 2 x 2 size와 stride 2로 연산됨   
  - 이렇게 완성된 Convolution layer들은 모델의 구조에 따라 그 개수가 다르며, 이름은 Layer의 깊이에 따라 VGG16, VGG19와 같이 정의됨   

&nbsp;

<div align="center">
  <img src="/assets/images/Net/2.png" width="40%" height="40%" alt=""/>
  <p><em></em></p>
</div>

&nbsp;

- **Fully Connected Layer**  
  - Convolution layer를 모두 정의하고 쌓아올린 후, 3개의 FC Layer가 사용됨   
  - 처음 2개의 Layer는 4096개의 채널을 가지고 있으며 마지막은 1000개의 채널로 구성됨  
    - 이는 최종적으로 분류해야 할 클래스가 1000개이기 때문   
  - 최종적으로 출력된 1000개의 값에 대해서 Softmax 연산을 수행   

&nbsp;

- **Hidden Layer**  
  - 모든 hidden layer에 대해서 ReLU Activation을 사용   
  - 또한 LRN 정규화 방법이 ImageNet Dataset에 대한 성능을 향상시키지 않고 오히려 메모리 사용량과 연산 시간의 증가만 초래   
    - 이는 AlexNet과 상반되는 주장

  - 각 convolution 연산을 수행할 때 진행되는 kernel의 채널 크기가 작은 것부터 시작됨   
    - 특히 각 Layer를 지나면서 kernel의 채널 크기는 512가 될 때까지 2배씩 증가

## 2. Discussion   

&nbsp;

- **Receptive Field**    
  - 저자가 주장한 ConvNet은 다른 모델들과 비교하면 kernel size, 즉 Receptive field를 훨씬 작게 설정함    
    - 다른 모델들이 11 x 11 혹은 7 x 7 size를 사용했었던 반면, 본 모델에서는 최소 크기에 가까운 1 x 1이나 3 x 3을 사용     
  - 어떻게 보면 3 x 3 size의 kernel을 2번 연속 사용하는 것은 5 x 5 kernel을 사용하는 것과 같고, 3 x 3을 3번 연속 사용하는 것은 7 x 7 kernel을 사용하는 것과 같을 수 있음, 이는 작은 size의 kernel을 연속적으로 사용할 때 다음과 같은 효과를 얻을 수 있기 때문    
    - Decision function을 더 차별적이게 만듦   
    - Parameter의 갯수를 감소시킬 수 있음    

&nbsp;

- **1 x 1 Kernel**   
  - 앞서, 1 x 1 size kernel은 입력 채널에 대한 선형 변환을 위해 사용된다고 해석함  
  - 저자는 이를 인접한 Receptive field에 영향을 주지 않으면서 비선형성을 증가시킬 수 있다고 설명함   

&nbsp;

## 3. Training  

&nbsp;

- **학습 방법**   
  - 학습 방법 자체는 AlexNet(2012)와 동일함   
    - 다중 로지스틱 회귀 분석   
    - 256-size Mini batch  
    - Momentum(0.9)     
  - 학습은 ***weight decay**를 사용하여 규제를 걸었으며, **drop out**은 초기 2개의 FC Layer에 적용함   
    - 이때 사용된 drop out rate는 0.5이며, Learning rate는 초기에 0.01로 설정함   
    - Validation accuracy가 saturation되었다면 Learning rate에 0.1을 곱하여 감소시키도록 설정    
  - 최종적으로 Learning rate는 3번 감소하였으며, 74 Epoch 동안 학습을 진행    
    - 더 적은 수의 parameter와 더 깊은 layer가 있음에도 학습 진행 횟수가 적었는데, 이는 깊은 layer의 수와 작은 convolution 연산이 정규화와 같은 역할을 가지고 있다고 저자들은 추정함    

&nbsp;

- **Weight Parameter 초기화**   
  - 각 네트워크에 대한 weight parameter의 초기화가 중요한데, 저자는 위 구조에서 가장 기본이 되는 A 모델 구조(11 Layer)를 학습시키고, 이 학습된 weight parameter를 다른 구조의 parameter를 초기화하는데 사용함    
  - 모든 Layer에 다 적용시킨 것은 아니고 처음 4개의 convolution layer와 마지막 3개의 FC Layer에만 초기화를 적용함    
  - 나머지 Layer에 대해서는 정규분포(0, 0.01)에서 랜덤한 수를 선택해 초기화를 진행

## 4. Testing  

&nbsp;

<div align="center">
  <img src="/assets/images/Net/6.png" width="40%" height="40%" alt=""/>
  <p><em></em></p>
</div>

&nbsp;

- **ConvNet**   
  - 테스트 과정에서는 입력된 이미지가 학습된 ConvNet을 거치는데, 이때 입력 이미지는 별도의 처리 과정을 거침   
  - 입력 이미지에 대해 미리 정해둔 size($$Q$$)로 재설정하는데 이때 원래의 비율을 유지하도록 함   
    - 학습 과정에서도 마찬가지로 size를 재설정하는 과정이 있는데, 이때의 크기를 $$S$$라고 한다면 $$Q$$와 $$S$$는 같을 필요는 없음   

&nbsp;

- **Deep Learning Layer**   
  - 위에서 size 조정을 마친 이미지는 마지막 3개의 Layer로 진입하는데, 이때 학습 과정에서 사용되었던 FC Layer들은 Convolution Layer로 대체되며 각각의 Layer는 다음과 같은 size로 대체됨    
    - 1st FC Layer → 7 x 7 Convolution Layer  
    - 2nd FC Layer → 1 x 1 Convolution Layer  
    - 3rd FC Layer → 1 x 1 Convolution Layer  
  - 이처럼 Test 과정에서는 모든 Layer가 Convolution 연산으로 바뀌었기 때문에 논문에서는 이를 **Fully Convolutional Network**라고 정의함    
    - 기존에는 하나의 이미지에서 여러 부분을 뽑아내서 augmentation하는 과정이 동반되었는데, 이는 계산량 증가를 초래하는 이슈가 있지만 본 논문에서는 이를 해소함   

&nbsp;

- **결과 산출**   
  - 모든 Layer를 거친 결과는 분류하고자 하는 클래스의 수와 같은 size를 지닌 채널이 될 것이며, 각 채널에 대해 속할 점수가 매겨져 있음   
  - 마지막으로 클래스 점수에 대한 고정 크기의 vector를 구하기 위해 산출할 클래스 점수들로 sum pooling을 진행함    
  - 입력 이미지에 대해서는 training과 마찬가지로 좌우반전을 통해 augmentation을 진행했고, Soft-max를 거친 결과에 대해 평균값을 구해 입력 이미지에 대한 최종 점수를 산출함
